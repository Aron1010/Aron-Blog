# Linux 入门：目录结构、路径与终端基本认知

## Linux 的目录结构

Linux 只有一个顶级目录，也就是根目录 `/`。所有文件、目录、设备、配置都挂在这棵目录树下面。

常见目录可以先这样理解：

| 目录 | 作用 |
| --- | --- |
| `/` | 根目录，Linux 文件系统的起点 |
| `/home` | 普通用户的家目录所在位置 |
| `/root` | root 用户的家目录 |
| `/etc` | 系统和软件配置文件 |
| `/var` | 日志、缓存、运行时数据等经常变化的文件 |
| `/usr` | 用户级软件、命令、库文件 |
| `/bin`、`/sbin` | 常用系统命令 |
| `/tmp` | 临时文件目录 |

例如：

```bash
/home/user/a.txt
```

表示从根目录 `/` 开始，依次进入 `home` 目录、`user` 目录，最后找到 `a.txt` 文件。

## HOME 目录

每个用户在 Linux 中通常都有自己的专属工作目录，也叫 HOME 目录。

- 普通用户的 HOME 目录通常是 `/home/用户名`
- root 用户的 HOME 目录是 `/root`
- 终端刚登录后，默认通常会进入当前用户的 HOME 目录

可以使用 `pwd` 查看当前位置：

```bash
pwd
```

如果当前登录用户是 `aron`，输出可能是：

```bash
/home/aron
```

## 绝对路径和相对路径

路径是 Linux 中非常重要的概念。它分为绝对路径和相对路径。

绝对路径从根目录 `/` 开始描述位置：

```bash
/home/aron/test/a.txt
```

不管你当前在哪个目录，这个路径指向的位置都固定不变。

相对路径不以 `/` 开头，它以当前所在目录作为起点：

```bash
test/a.txt
```

如果当前目录是 `/home/aron`，那么 `test/a.txt` 实际指向：

```bash
/home/aron/test/a.txt
```

## 特殊路径符

Linux 中有几个很常见的特殊路径符。

| 符号 | 含义 | 示例 |
| --- | --- | --- |
| `.` | 当前目录 | `./a.txt` |
| `..` | 上一级目录 | `cd ..` |
| `../..` | 上两级目录 | `cd ../..` |
| `~` | 当前用户 HOME 目录 | `cd ~` |

例如，回到用户 HOME 目录：

```bash
cd ~
```

进入上一级目录：

```bash
cd ..
```

执行当前目录下的脚本：

```bash
./start.sh
```

这里的 `./` 表示“当前目录下”。

## 隐藏文件和隐藏目录

Linux 中，以 `.` 开头的文件或目录默认是隐藏的，例如：

```bash
.bashrc
.ssh
.vimrc
```

普通 `ls` 看不到隐藏文件，需要加 `-a`：

```bash
ls -a
```

很多用户配置、软件配置、SSH 密钥都会放在隐藏文件或隐藏目录里，所以排查环境变量、登录配置、密钥问题时经常会用到 `ls -a`。

## 查看命令帮助

查看简要帮助：

```bash
命令 --help
```

例如：

```bash
ls --help
```

查看详细手册：

```bash
man 命令
```

例如：

```bash
man ls
```

在 `man` 手册中按 `q` 可以退出。

## 本篇命令速查

| 命令 | 作用 |
| --- | --- |
| `pwd` | 查看当前工作目录 |
| `cd 目录` | 切换目录 |
| `cd ~` | 回到当前用户 HOME 目录 |
| `cd ..` | 回到上一级目录 |
| `ls -a` | 查看包含隐藏文件在内的目录内容 |
| `命令 --help` | 查看命令简要帮助 |
| `man 命令` | 查看命令详细手册 |
