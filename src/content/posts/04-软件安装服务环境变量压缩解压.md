# Linux 软件安装、服务管理、环境变量与压缩解压

## 软件安装方式概览

Linux 软件安装常见方式有三类：

| 方式 | 常见系统 | 特点 |
| --- | --- | --- |
| `rpm` | CentOS、RHEL、openEuler 等 | 安装本地 `.rpm` 包，依赖需要额外处理 |
| `yum` / `dnf` | CentOS、RHEL、Fedora 等 | 自动处理依赖，适合在线安装 |
| `apt` | Ubuntu、Debian 等 | 自动处理依赖，Ubuntu 最常用 |

入门阶段建议优先掌握 `yum` 和 `apt`，因为它们会自动处理大部分依赖问题。

## rpm：本地软件包管理

安装 rpm 包：

```bash
rpm -ivh 软件包名.rpm
```

升级 rpm 包：

```bash
rpm -Uvh 软件包名.rpm
```

查询软件是否安装：

```bash
rpm -q 软件包名
```

查询所有已安装软件包：

```bash
rpm -qa
```

查询软件包详细信息：

```bash
rpm -qi 软件包名
```

查询软件安装了哪些文件：

```bash
rpm -ql 软件包名
```

查询某个文件属于哪个 rpm 包：

```bash
rpm -qf /usr/bin/vim
```

`rpm` 的缺点是依赖处理不够方便。比如安装 A 包时，如果它依赖 B、C 包，就需要手动补齐依赖。

## yum：CentOS 常用软件管理

安装软件：

```bash
yum install 软件名
yum install -y 软件名
```

`-y` 表示自动确认。

卸载软件：

```bash
yum remove 软件名
```

搜索软件：

```bash
yum search 关键字
```

查看软件信息：

```bash
yum info 软件名
```

列出可安装和已安装软件：

```bash
yum list
```

检查可更新软件：

```bash
yum check-update
```

更新全部软件：

```bash
yum update
```

清理缓存：

```bash
yum clean all
```

配置本地 yum 源时，常见步骤是挂载光盘并配置 repo 文件：

```bash
mount /dev/sr0 /mnt
vi /etc/yum.repos.d/local.repo
```

示例内容：

```ini
[local]
name=centos7
baseurl=file:///mnt
gpgcheck=0
```

查看 yum 源：

```bash
yum repolist
```

## apt：Ubuntu 常用软件管理

Ubuntu 和 Debian 系统常用 `apt`。

更新软件源缓存：

```bash
sudo apt update
```

安装软件：

```bash
sudo apt install 软件名
```

卸载软件：

```bash
sudo apt remove 软件名
```

搜索软件：

```bash
apt search 关键字
```

升级系统已安装软件：

```bash
sudo apt upgrade
```

无论是 `yum` 还是 `apt`，安装和卸载系统软件通常都需要 root 权限。

## wget：命令行下载文件

`wget` 是非交互式下载工具，适合在服务器上下载文件。

```bash
wget URL
```

后台下载：

```bash
wget -b URL
```

后台下载日志默认写入当前目录的 `wget-log`，可以查看进度：

```bash
tail -f wget-log
```

下载并指定保存文件名：

```bash
wget -O CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo
```

## systemctl：系统服务管理

`systemctl` 用来管理系统服务，比如 nginx、mysql、sshd、ntpd。

```bash
systemctl start 服务名
systemctl stop 服务名
systemctl restart 服务名
systemctl status 服务名
systemctl enable 服务名
systemctl disable 服务名
```

含义：

| 命令 | 作用 |
| --- | --- |
| `start` | 启动服务 |
| `stop` | 停止服务 |
| `restart` | 重启服务 |
| `status` | 查看服务状态 |
| `enable` | 设置开机自启 |
| `disable` | 关闭开机自启 |

示例：

```bash
systemctl status sshd
systemctl restart nginx
systemctl enable nginx
```

排查服务问题时，通常先看状态，再看日志：

```bash
systemctl status nginx
journalctl -u nginx
```

## 软链接：ln -s

软链接类似快捷方式。

```bash
ln -s 源文件或目录 链接位置
```

示例：

```bash
ln -s /usr/local/nginx/sbin/nginx /usr/bin/nginx
```

这样就可以直接执行 `nginx`，不用每次输入完整路径。

查看软链接：

```bash
ls -l
```

软链接前面通常会显示 `->` 指向原路径。

## 日期、时区和时间同步

查看当前时间：

```bash
date
```

格式化输出：

```bash
date "+%Y-%m-%d"
date "+%Y-%m-%d %H:%M:%S"
```

常见格式：

| 格式 | 含义 |
| --- | --- |
| `%Y` | 四位年份 |
| `%y` | 两位年份 |
| `%m` | 月 |
| `%d` | 日 |
| `%H` | 小时 |
| `%M` | 分钟 |
| `%S` | 秒 |
| `%s` | Unix 时间戳 |

修改时区可以使用：

```bash
timedatectl set-timezone Asia/Shanghai
```

安装 NTP：

```bash
yum install -y ntp
```

启动 NTP 服务：

```bash
systemctl start ntpd
systemctl enable ntpd
```

手动校准时间：

```bash
ntpdate -u ntp.aliyun.com
```

## 环境变量和 PATH

查看全部环境变量：

```bash
env
```

查看某个变量：

```bash
echo $PATH
```

临时设置变量：

```bash
export APP_HOME=/opt/app
```

临时变量只在当前终端有效。想永久生效，可以写入：

- 用户级配置：`~/.bashrc`
- 全局配置：`/etc/profile`

`PATH` 记录了系统查找可执行命令的路径。比如把自定义脚本目录加入 PATH：

```bash
export PATH=$PATH:/opt/scripts
```

如果变量名后面要紧跟其他字符，建议使用 `${}`：

```bash
echo ${PATH}ABC
```

## 压缩和解压

使用 `tar` 打包压缩：

```bash
tar -zcvf archive.tar.gz file1 dir1
```

常用选项：

| 选项 | 作用 |
| --- | --- |
| `-z` | 使用 gzip |
| `-c` | 创建归档 |
| `-v` | 显示过程 |
| `-f` | 指定文件名 |
| `-t` | 查看归档内容 |
| `-x` | 解包 |
| `-C` | 指定解压目录 |

查看压缩包内容：

```bash
tar -tf archive.tar.gz
```

解压：

```bash
tar -zxvf archive.tar.gz
tar -zxvf archive.tar.gz -C /opt
```

使用 `zip` 压缩：

```bash
zip file.zip a.txt
zip -r project.zip project
```

使用 `unzip` 解压：

```bash
unzip file.zip
unzip file.zip -d /opt/project
```
